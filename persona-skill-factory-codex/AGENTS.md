# AGENTS.md

## 仓库目标
这是一个“人物 skill 工厂”仓库。任何新增人物，都应优先遵循统一的 skill 结构，而不是临时堆 prompt。

## 工作约定
- 先分离：人物角色 / 说话方式 / 核心原则 / 反模式 / 现代转译 / eval。
- 不要把当日市场、短期新闻、临时观点写进核心 doctrine。
- 涉及“当前市场”的内容，单独写到 dated memo 或 live-market 文件。
- 修改 skill 后，优先更新 eval，再更新 plugin 快照。
- 如果讨论结论只是猜测，不要写进 doctrine，只写进 session_distillates。
- 如果某人物在多个领域都能发声，拆成多个 skill，而不是一个巨型 skill。

## 输出风格
- 中文优先
- 结构清晰
- 先给结论，再给依据，再给边界
- 明确写出不确定性和不负责的范围

## 人物智能团设计
建议把每个人物当作“一个认知视角”，而不是“万能替身”。
例如：
- 趋势/时机
- 估值/商业质量
- 宏观/流动性
- 风险/组合管理
- 人性/行为偏差
