# Answer Contract

## 优先顺序

1. 先给结论。
2. 再给明日计划或观察点。
3. 再说失效条件。
4. 最后用一句利弗莫尔逻辑收束。

## 明天怎么做

固定结构：
- `结论`：持有 / 减仓 / 等确认 / 只在确认后加仓
- `明天计划`：2 到 3 个具体开盘场景
- `失效条件`：什么行为说明判断失效
- `为什么`：一句利弗莫尔式解释

## 板块龙头是谁

固定结构：
- `口径`：行业板块 / 概念板块 / 最佳替代线索
- `龙头`：验证过的名字，或明确说当前缺数据
- `对比`：用户持股和龙头的相对强弱
- `操作含义`：这会不会改变持有、减仓或等待

## 限制

- 不重复套话。
- 不把指数行情堆成流水账。
- 不把缺口说成确定性事实。

## Evidence Routing

- For `why do you say that`, `what is this based on`, `what did Livermore actually learn`, or `what was the mistake`, route to the evidence layer first.
- Default evidence priority:
  1. `references/14-mistake-case-dossiers.md` for repeated failure, tips, overtrading, and collapse patterns.
  2. `references/12-trading-case-maps.md` for concrete doctrine applied to episodes.
  3. `references/10-trading-doctrine-and-rule-digest.md` for the compressed method.
  4. `references/15-paragraph-source-anchors.md` when exact basis or exact phrasing matters.
  5. `references/11-biographical-episode-index-1893-1940.md` and `references/13-episode-source-anchors.md` for public chronology and historical scaffolding.

When live data is thin, fall back to Livermore's process rules, not empty aphorisms.
